# Phase 73 — Admin Console & shared app shell

> midnite has every operator-facing surface it needs *as data* — [Phase 61](phase-61-fable-observability.md) gives real token/cost **usage attribution** + cycle-time metrics, [Phase 22](phase-22-fleet-visibility.md) the runtime **Ops** metrics, [Phase 33](phase-33-multi-user-teams.md) **users/teams/invites** + audit, [Phase 55](phase-55-projects-detail-page.md) the **projects** cockpit, and [Phase 71](phase-71-app-update-banner.md) the build-emitted **version manifest** + changelog + channels/floor. What's missing is a single **operator-facing app** that composes them: a standalone **`packages/admin`** Next.js console — platform usage, users, projects, versions, quick links — that **looks and feels exactly like `web`**. Phase 73 delivers it by first extracting the reusable app shell into a **new `@midnite/shell`** package (and the pure visuals into `@midnite/ui`), refactoring `web` to consume it so there's one source of truth, then standing up `admin` on the same shell behind a new **operator** identity — including a **lock screen** (idle re-lock **and** themed login) reusing the neuro-cloud **starfield/dots**.

> **Scope guardrails (CLAUDE.md).** This introduces **two new packages** and therefore **two new boundary edges** — get them right: `@midnite/ui` stays a **strict leaf** (only pure visuals move in; the `BackgroundPattern` union is **inlined** so `ui` still imports nothing in-repo, keeping [`boundary.test.ts`](../packages/ui/src/boundary.test.ts) green), and **`@midnite/shell`** is a new mid-tier package that **may depend on `shared` + `ui`** (with `next` + `@tanstack/react-query` as **peers**) but **never** on `web`/`admin`/`gateway` internals. `admin`, like `web`, is a **pure HTTP/WS client of the gateway** — it never imports gateway internals; cross-package types live in `shared`. **`shared` remains the config + contract authority.** The operator gate rides the [Phase 72](phase-72-sso-go-live-operator-config.md) operator-config seam — no new persisted global-role model. Versions is **view-only** — no new gateway write API.

> Effort tags: **S** small · **M** medium · **L** large. Critical path is **A → B → C** (extract visuals → build the shell → refactor `web` onto it, proving the shell is real and web is unbroken) then **D → E** (operator gate → scaffold `admin` on the shell); **F** (the admin sections) is the payload and **G** (tests/docs/boundaries) hardens. A+B+C is "the shell exists and web still works"; D+E is "the admin app boots, gated"; F is "operators can actually see everything".

- **The two real gaps** — (1) **no cross-user/cross-team "list all" admin endpoint** ([`users.controller.ts`](../packages/gateway/src/users/users.controller.ts) is self-only `GET/PATCH /users/me`; [`admin.controller.ts`](../packages/gateway/src/admin/admin.controller.ts) is just `POST /admin/backup`); (2) **no global operator role** — `TeamRole` (`owner/admin/member/viewer`, [`shared/src/team.ts`](../packages/shared/src/team.ts)) is **team-scoped only**, enforced by [`role.guard.ts`](../packages/gateway/src/auth/role.guard.ts); usage/metrics/audit carry **no** role guard, and static-token/anon installs bypass role checks entirely. Version/changelog is a **static** web-served `version.json` polled by [`lib/version.ts`](../packages/web/lib/version.ts) — **not** a gateway API.
- **Operator-config seam (Phase 72)** — [Phase 72](phase-72-sso-go-live-operator-config.md) carves `gateway.auth` into a gitignored **`.midnite/operator.json`** via a reusable `OperatorConfigSchema` (deep-merged in [`config-loader.ts`](../packages/shared/src/config-loader.ts)). Phase 73's operator allowlist rides the **same file** — the natural home for "who is an operator".

## Decisions / open questions

1. **Separate app vs. section of web** → **Resolved (user): a standalone `packages/admin` app**, not a `/admin` route group — with the shell extracted so both apps genuinely share it.
2. **Where the reusable shell lives** → **Resolved (user): a new `@midnite/shell` package** (Standard depth). Pure visuals go to leaf `@midnite/ui`; the wired `<AppFrame>` + lock + appearance runtime + providers go to `shell` (which may depend on `shared`+`ui`). The wired shell **cannot** live in leaf `ui` — importing `shared`/data would break its boundary test. `web` is refactored to consume `<AppFrame>` so there's one source of truth.
3. **Shell depth** → **Resolved (user): Standard** — `<AppFrame>` (injected nav config) + lock screen + appearance runtime + provider stack. The **settings framework stays web-owned** (admin reuses only appearance + screen-lock), keeping the web refactor bounded.
4. **Operator gate** → **Resolved (user): an operator allowlist in the Phase 72 operator-config** (`.midnite/operator.json`), deriving an `isOperator` claim + `@RequiresOperator` guard. **No** new persisted global-role model. Soft-depends on Phase 72's `OperatorConfigSchema`; if 72 hasn't landed, Phase 73 introduces the minimal operator-config read path it needs.
5. **Existing usage/metrics/audit gating** → **Recommendation: leave them as-is.** Admin's platform-wide views use **new** operator-gated aggregate endpoints so we don't risk breaking web's Ops page (which reads the ungated team-scoped routes). *(Recommended; confirm in review.)*
6. **Versions scope** → **Resolved (user): view-only** — released versions + changelog + channels/floor read-only + quick links. Managing channels/floor or publishing releases (needs a new gateway version-manifest **write** API) is **out of scope**.
7. **Lock screen** → **Resolved (user): both** an idle+passcode re-lock **and** a themed login gate, both on the neuro-cloud starfield/dots, housed in `@midnite/shell`.
8. **Admin sections** → **Resolved (user): Core + Audit + Teams management** — Overview, Usage & cost, Users & teams (list + team CRUD/role mgmt), Projects, Versions (view-only), Audit log, Quick links. Spend caps (Phase 50) + service tokens (Phase 38) mirrors are **out of scope** for this phase (a later widen).
9. **Admin login mechanism** → **Recommendation: reuse Phase 70 SSO** for the login gate + the operator allowlist check; passcode only for the idle re-lock. *(Recommended; confirm in review.)*
10. **Is `admin` packaged into desktop?** → **Recommendation: no** — a web-only standalone deployable for now (like a hosted operator console), not bundled into the Electron app. *(Recommended; confirm in review.)*
11. **`site-links` home** → if quick links are shared by `web` + `admin`, lift [`lib/site-links.ts`](../packages/web/lib/site-links.ts) into `shared` (or `shell`); otherwise duplicate the small list. *(Lean shared.)*
