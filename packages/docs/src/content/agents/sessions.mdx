# Sessions

## What it is

A **session** is one agent run, wrapped around an embedded terminal. There's one
session per [task](/app/tasks), and its status mirrors that task's — so a session
is really the *execution view* of a card on the board. The screen lists two kinds:

- **Live runs** — currently executing, with a real terminal you can watch stream
  in real time.
- **Past runs** — finished, each with a full **transcript** you can read end to
  end.

## Purpose

Sessions are your window into what an agent is *actually doing*, keystroke by
keystroke. The [board](/app/tasks) tells you a task is running; the session shows
you how. Open the live terminal to watch a run unfold, read the transcript
afterwards to audit every step it took, and archive the runs worth keeping for
reference. It's the detail layer beneath the board — and the same stream the
[Office](/overview/office) floor lets you walk up to.

## Creating one

Sessions are usually created for you when a task starts, but you can also launch
one directly.

| Action | How | Behind the scenes |
| --- | --- | --- |
| Launch a session | **New session** → launch modal | session launch |

## Listing & viewing

| Action | How | Behind the scenes |
| --- | --- | --- |
| List sessions | Sessions screen | `getSessions` |
| Open a session | Click it | `getSession` → `/sessions/view` |
| Open the live terminal | **Terminal** on a live session | `mintTerminalToken` |
| Read the transcript | **Transcript** on a session | `getSessionTranscript` |

## Managing

| Action | How | Behind the scenes |
| --- | --- | --- |
| Archive | Session menu → **Archive** | `archiveSession` |
| Unarchive | Session menu → **Unarchive** | `unarchiveSession` |
| Delete | Session menu → **Delete** | `deleteSession` |

> The live terminal is a real PTY stream (xterm.js). The short-lived token minted
> by `mintTerminalToken` authorises the WebSocket connection to it.
