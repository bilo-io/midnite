# Report an issue

Found a bug? Open the **assistant menu** — the logo button in the bottom-right
corner — and pick **Report issue**. It turns "something's broken" into a
ready-to-file GitHub issue in a couple of clicks.

## What happens

1. A preview opens with a **title** and a **details** box that's already filled in
   with a short description prompt and the context of the page you were on:
   the route, the app version, your browser and OS, whether you're on the web or
   the desktop app, your theme, and the live-connection status.
2. **Edit anything.** The whole preview is editable — write what happened at the
   top, and delete any context line you'd rather not share. The issue is
   **public**, so nothing leaves your machine until you choose to open it.
3. Click **Open on GitHub**. Your browser opens a new GitHub issue with everything
   prefilled — review it once more and submit.

## Privacy

The page context is gathered entirely in your browser and is never sent anywhere
except into the GitHub issue you open yourself. Because the details box is fully
editable, you're always in control of exactly what's included.

If a report is unusually long, the preview trims the auto-captured environment
details to fit GitHub's link limit and offers a **Copy body** button so you can
paste the full text into the issue after it opens.
