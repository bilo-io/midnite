# midnite

**midnite is a desktop app for running many coding agents at once.** You describe
the work as tasks; midnite schedules them across a pool of Claude Code agents,
runs each in its own session, and shows the whole fleet on a live kanban board —
so you can supervise a dozen agents the way you'd supervise a team, instead of
babysitting one terminal.

It's a single download that runs entirely on your machine. Nothing to host, no
server to stand up — install it and open the app.

## What you can do with it

The sidebar groups midnite's features the same way the app does:

- **App** — the day-to-day surfaces: the [Tasks](/app/tasks) board that schedules
  agent work, [Projects](/app/projects) that plan it, [Workflows](/app/workflows)
  that automate it, and [Slides](/app/slides) for quick decks.
- **Agents** — everything about the agents themselves: live and past
  [Sessions](/agents/sessions), long-term [Memory](/agents/memory) you inject into
  their prompts, multi-agent [Councils](/agents/councils), and a
  [Media](/agents/media) library for what they generate.
- **Overview** — the fleet-wide views: the [Dashboard](/overview/dashboard), the
  pixel-art [Office](/overview/office) floor, and [Ops](/overview/ops) analytics
  with its periodic [Digests](/overview/digests).
- **Settings** — where you [configure](/settings/settings) the agent CLI, your
  repos, your team, and the safety guardrails.

## How it fits together

Under the hood, midnite is a small local **gateway** with thin clients in front of
it. The gateway is the brain — it holds your tasks in a local SQLite database,
runs the scheduler, and manages a pool of agent processes. The **desktop app** (the
web UI you see) and the optional **CLI** both talk to that gateway over HTTP and
WebSocket on `localhost`; the gateway spawns each coding agent in its own pseudo-
terminal.

<ArchitectureDiagram />

Because everything runs locally, your code, prompts, and task history stay on your
machine — the only things that leave are the agents' own calls to their model
provider.

## The command line

The desktop app is the main way in, but midnite also ships a terminal client,
`midnite`, for people who live in the shell or want to script their fleet. It
talks to the same local gateway, so the board, the CLI, and any agent all see one
source of truth.

<Terminal title="midnite --help" content={cliHelp} />

`midnite watch` opens a full-screen live dashboard right in your terminal — the
board, the agent pool, and streaming logs — and `midnite serve` runs the gateway
on its own if you'd rather not keep the desktop app open.
