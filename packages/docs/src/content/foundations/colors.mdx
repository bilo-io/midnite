# Colours

The palette is **semantic**, not literal: components reference roles (`primary`,
`muted`, `destructive`, …) rather than raw hues, so a single token edit re-themes
everything. Each role is an HSL custom property with a light value and a `.dark`
override — the swatches below paint with `hsl(var(--token))`, so they reflect the
**active theme**. Flip the switcher in the header to watch them change.

## Using a token

In Tailwind, every role maps to a utility (`bg-primary`, `text-muted-foreground`,
`border-destructive`, …); the mapping lives in your `tailwind.config.ts`. In plain
CSS or inline styles, consume it directly:

```css
.thing {
  background: hsl(var(--primary));
  color: hsl(var(--primary-foreground));
}
```

Foreground tokens are paired with their surface (e.g. `primary` / `primary-foreground`)
so text always has the right contrast on that surface.
