# Radius & scales

## Border radius

Radii derive from a single `--radius` base (`0.5rem`), the shadcn convention: `lg` is
the base, `md` and `sm` step down from it. Mapped to Tailwind's `rounded-lg` /
`rounded-md` / `rounded-sm`.

## Reserved scales

Several scales are **documented placeholders** in the token map — the app uses
Tailwind's defaults for them today, and they're reserved (clearly marked
`placeholder: true`) until the design system formalises its own:

| Scale | Status | Today |
| --- | --- | --- |
| `spacing` | placeholder | Tailwind's default rem scale |
| `shadow` | placeholder | Tailwind's default elevation |
| `motion` | placeholder | Tailwind's default durations / easing |
| `zIndex` | partial | a few named stacking levels (`dropdown`, `tooltip`) |

Keeping them in the token map — even empty — makes the taxonomy explicit and the
system obviously extensible.
