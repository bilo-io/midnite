# Configuration (`midnite.json`)

All user configuration lives in a single `midnite.json` at the project root,
validated by one zod schema in
[`packages/shared/src/config.ts`](https://github.com/bilo-io/midnite/blob/main/packages/shared/src/config.ts).
That schema is the **source of truth** — this page documents it, but if the two
ever disagree, the schema wins. Every field below is read through
`shared`'s `loadConfig()`; no consumer reads `midnite.json` directly.

Almost every field has a default, so a minimal config is small. The `agent`,
`terminal`, and `gateway` keys are expected (pass `{}` to accept all their
defaults); everything else is fully optional and defaults to a sensible value.

```json
{
  "agent": { "pool": 4, "poolEnabled": true },
  "terminal": { "mode": "tmux" },
  "gateway": { "port": 7777 },
  "repos": [{ "name": "midnite", "path": "~/Dev/midnite" }]
}
```

> **Secrets are never inlined.** Tokens and keys are referenced by the *name* of
> an environment variable (`tokenEnv`, `clientSecretEnv`, `encryptionKeyEnv`),
> not written into committed config.

## `agent`

The agent pool: how many sessions run, which models, and the scheduler's
behaviour.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `pool` | int > 0 | `4` | Number of agent slots. |
| `provider` | `anthropic` \| `openai` \| `google` \| `openai-compatible` | `anthropic` | Default LLM provider for the gateway's own AI features. DB provider settings are the runtime source of truth; this is the fallback. (`claude` is normalised to `anthropic`.) |
| `plan` | string | `opus4.8` | Model used for planning. |
| `act` | string | `haiku4.5` | Model used for acting. |
| `poolEnabled` | boolean | `false` | Feature flag for the scheduler. When off, sessions only spawn when a human attaches a terminal. |
| `schedulerTickMs` | int > 0 | `5000` | How often the pool tick loop wakes to assign ready `todo` tasks to free slots. |
| `waitingHoldsSlot` | boolean | `true` | Whether a `waiting` task keeps its pool slot (its PTY is still alive, blocked on stdin). |
| `runTimeoutMs` | int > 0 | `1800000` | Hard ceiling per autonomous run; the session is cancelled on expiry and the task requeued. |
| `maxRetries` | int ≥ 0 | `3` | Auto-retries after an unexpected agent exit (crash) before abandoning. `0` = never. |
| `maxPerRepo` | int ≥ 0 | `0` | Max concurrent agents on the same repo (keyed by `task.repo`). `0` = unlimited. |

## `terminal`

The process backend for agent/interactive sessions.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `mode` | `pty` \| `tmux` | `pty` | `pty` dies with the gateway; `tmux` runs each session detached so an in-flight run survives a restart (reattached on boot). |
| `layout` | `split` \| `tabs` \| `windows` | `split` | Multi-session layout hint. |
| `command` | string | _(login shell)_ | Command spawned for an on-demand session PTY. |
| `args` | string[] | `[]` | Arguments for `command`. |
| `scrollbackBytes` | int > 0 | `262144` | Bytes of recent output retained per PTY for scrollback replay on (re)attach. |
| `idleDisposeMs` | int ≥ 0 | `300000` | Grace period after the last client detaches before the PTY is reaped. |
| `maxSessions` | int > 0 | `16` | Max concurrent live PTYs; further spawns are rejected until one frees. |
| `inheritSecrets` | boolean | `false` | Pass the gateway's secret-looking env vars (API keys) into the PTY. Enable for `command: "claude"`. |
| `hookCallbackUrl` | string | _(loopback)_ | URL the in-PTY hook script calls back on. Defaults to the gateway's loopback address. |

### `terminal.approvals`

Human-in-the-loop tool approvals for `command: "claude"` sessions — a PreToolUse
hook routes Claude's permission requests to the web UI.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | boolean | `false` | Only meaningful when the PTY command is `claude`. |
| `timeoutMs` | int > 0 | `120000` | How long the gateway holds a pending approval before auto-resolving. |
| `onTimeout` | `deny` \| `ask` | `deny` | Action on timeout (fail-safe = deny). |
| `onNoSubscriber` | `deny` \| `ask` | `ask` | Action when no browser is watching (fall back to Claude's own TUI prompt). |

## `repos`

An array seeding the repo registry on first boot; the DB is authoritative
thereafter.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `name` | string | _(required)_ | Repo display name. |
| `path` | string | _(required)_ | Absolute or `~`-relative path to the working tree. |
| `branchPrefix` | string | _(optional)_ | Default branch prefix, fed to the agent's seed prompt. |
| `prTemplate` | string | _(optional)_ | PR template seeded into the agent's seed prompt. |

## `gateway`

The HTTP/WS server.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `port` | int > 0 | `7777` | Listen port. |
| `host` | string | `127.0.0.1` | Bind address. Loopback by default — the gateway spawns PTYs, so don't expose it unless you mean to. |
| `allowedOrigins` | string[] | `[]` | Extra browser origins allowed to call the API / open the terminal WS. Loopback is always allowed. |
| `uploadsDir` | string | `./.midnite/uploads` | Upload storage directory. |
| `dbPath` | string | `./.midnite/midnite.db` | SQLite database path. |
| `webDir` | string | _(optional)_ | Path to the web app's static export (`packages/web/out`). When set with an `index.html`, the gateway serves the UI at `/`. Override at runtime with `MIDNITE_WEB_DIR`. |

### `gateway.auth`

Optional remote-access auth — off by default (midnite is local-only out of the
box).

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `tokenEnv` | string | `MIDNITE_AUTH_TOKEN` | Name of the env var holding the bearer token. When set (non-empty), every REST route requires `Authorization: Bearer <token>` except `/health` and `/hooks/*`. |
| `requireOnNonLoopback` | boolean | `true` | Fail-closed: refuse to boot when binding a non-loopback host with no token. Set `false` to bind non-loopback intentionally without a token. |

#### `gateway.auth.rateLimit`

Basic per-IP fixed-window rate limit.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `windowMs` | int > 0 | `60000` | Window length. |
| `max` | int ≥ 0 | `0` | Requests per window per IP before a `429`. `0` disables it (the default). `/health` is never throttled. |

## `checks`

Quality-gate checks run before a task's `done` transition (Phase 30). Optional
and fully defaulted.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | boolean | `false` | Master switch for the gate. |
| `gates` | `Check[]` | `[]` | Checks applied to any repo by default. |
| `byRepo` | `record<string, Check[]>` | `{}` | Per-repo checks, keyed by repo name. |
| `autoFix.enabled` | boolean | `false` | Let an agent attempt to fix failing checks. |
| `autoFix.maxAttempts` | int > 0 | `2` | Max auto-fix attempts. |
| `perCheckTimeoutMs` | int > 0 | `600000` | Timeout per check when a `Check` sets no `timeoutMs`. |
| `outputCapBytes` | int > 0 | `16384` | Captured stdout+stderr is tail-truncated to ~this many bytes. |

A `Check` is `{ name, command, cwd?, timeoutMs? }` — `cwd` is repo-relative
(defaults to the repo root).

## `notifications`

State-transition notifications (Phase 21). The gateway watches transitions,
applies this policy, and dispatches to the enabled channels.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | boolean | `true` | Master switch. |
| `events.taskWaiting` | boolean | `true` | Notify when a task enters `waiting`. |
| `events.taskDone` | boolean | `true` | Notify when a task is `done`. |
| `events.taskAbandoned` | boolean | `true` | Notify when a task is `abandoned`. |
| `channels.web` | boolean | `true` | Always-on in-app feed. |
| `channels.browser` | boolean | `false` | Opt-in OS notification. |
| `channels.webhook` | string | _(optional)_ | Optional SSRF-guarded POST target. |

## `workflows`

The workflow engine (greenfield — ships off by default).

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | boolean | `false` | Feature flag. |
| `defaultTimezone` | string | `UTC` | Timezone applied to cron triggers that don't specify one. |
| `schedulerTickMs` | int > 0 | `30000` | How often the scheduler evaluates cron triggers. |
| `webhookBaseUrl` | string | `http://localhost:7777` | Base URL used to build copyable webhook URLs. |
| `allowLoopbackHttp` | boolean | `false` | Allow `http.request` nodes to call loopback hosts. Off by default (SSRF guard). Non-loopback private ranges stay blocked. |
| `encryptionKeyEnv` | string | `MIDNITE_WORKFLOWS_KEY` | Env var name for the credential-vault key. |
| `oauth.slack` / `oauth.google` | OAuth client | _(optional)_ | `{ clientId, clientSecretEnv, scopes[] }` per provider. |

## `agents`

Runtime knobs for the agent heartbeat feature (distinct from the `agent` pool).

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `heartbeatEnabled` | boolean | `false` | Feature flag for the heartbeat scheduler. |
| `schedulerTickMs` | int > 0 | `60000` | How often the heartbeat tick loop checks whether a run is due. |

## `councils`

Multi-agent council debates.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `runTimeoutMs` | int > 0 | `600000` | Hard ceiling per participant one-shot run; the PTY is killed on expiry (partial output retained). |

## `usage`

LLM usage/cost tracking with optional soft budgets. Budgets are **advisory** —
calls are never blocked.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `dailyBudgetUsd` | number > 0 | _(optional)_ | Soft daily budget; omit to disable the daily warning. |
| `monthlyBudgetUsd` | number > 0 | _(optional)_ | Soft monthly budget; omit to disable the monthly warning. |
| `warnAtRatio` | 0–1 | `0.8` | Fraction of a budget at which to start warning. |

## `knowledge`

A watched folder of Markdown the plan model can pull into an agent's execution
prompt.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | boolean | `false` | Feature flag. |
| `dir` | string | _(optional)_ | Directory of Markdown to watch. Inert when unset. `~`/relative paths resolved by the gateway. |
| `maxBytes` | int > 0 | `16384` | Byte cap on knowledge content injected into one prompt. |

## `prStatus`

Live PR-status polling (Phase 22). A gateway loop refreshes the GitHub
state/CI/review of tasks whose PR isn't merged/closed. Fail-open on any error.

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | boolean | `true` | Poll open PRs for live state. |
| `pollIntervalMs` | int > 0 | `60000` | How often the poller refreshes unmerged PRs. |
| `pollConcurrency` | int > 0 | `4` | Max PRs fetched concurrently per cycle (bounded for rate limits). |
