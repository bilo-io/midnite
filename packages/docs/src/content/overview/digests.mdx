# Digests

## What it is

**Fleet digests** — periodic roll-ups of what shipped, what failed, and what needs
attention across your workspace. Each digest is a snapshot of a time window,
generated for you; the screen is a mostly read-only feed of them.

## Purpose

Digests answer "what happened while I was away?" in one place. Rather than
reconstructing activity from the board and sessions, you read the summary — the
wins, the failures, and the follow-ups — and drill in only where you need to.

## Listing & viewing

| Action | How | Behind the scenes |
| --- | --- | --- |
| List digests | Digests screen | `getDigests` |
| Open a digest | Select one (`?id=`) | `getDigest` |
| Export as Markdown | Digest menu → **Export** | `exportDigest` |

> Digests are generated on a cadence rather than created by hand — the screen is a
> feed to read and export, not a CRUD surface.
