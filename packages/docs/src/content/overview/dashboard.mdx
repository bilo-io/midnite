# Dashboard

## What it is

The Dashboard is your at-a-glance **home surface** — a configurable grid of
widgets that summarises live status, clocks, and fleet activity in one place. It's
built around two ideas:

- **Widgets** are self-contained cards. Each one surfaces a slice of your
  workspace — the current time across zones, live agent activity, recent
  [workflow](/app/workflows) runs, or the latest [digest](/overview/digests) —
  and refreshes on its own.
- **Tabs** are independent layouts. Every tab has its own arrangement of widgets,
  so you can keep separate views side by side (one per team, one per timeframe, a
  focused "what's running now" board) and switch between them instantly.

## Purpose

The Dashboard is where you land to read the state of everything without drilling
into any single feature. Rather than hopping between the [board](/app/tasks),
[Sessions](/agents/sessions), and [Ops](/overview/ops), you compose the handful of
widgets that matter to you, pin the tab you use most, and adjust the timeframe to
zoom activity in or out. Think of it as the cockpit: a quick read on fleet health
before you decide where to dive in.

It's especially useful for:

- **Morning triage** — one glance to see what shipped overnight and what needs a look.
- **Focused sessions** — a stripped-down tab with only the widgets for the work in front of you.
- **Monitoring at scale** — a fleet-activity tab you leave open while a batch of agents runs.

## Tabs

Tabs let you keep several dashboards at once. Add as many as you like, pin the one
you open most so it leads the bar, and drag to reorder them.

| Action | How | Behind the scenes |
| --- | --- | --- |
| Add a tab | **+** on the tab bar | dashboard tabs |
| Rename a tab | Double-click the tab | dashboard tabs |
| Pin / unpin a tab | Tab menu | dashboard tabs |
| Reorder tabs | Drag on the tab bar | dashboard tabs |
| Close a tab | Tab menu → **Close** | dashboard tabs |

## Widgets

Compose each tab from the widget catalogue. Add the cards you care about, remove
the ones you don't, and use the **timeframe** control to rescale every
time-aware widget on the tab at once — from the last hour to the last month.

| Action | How | Behind the scenes |
| --- | --- | --- |
| Add a widget | **Add widget** | dashboard grid |
| Remove a widget | Widget menu → **Remove** | dashboard grid |
| Change timeframe | Timeframe control | dashboard timeframe |

> The root route (`/`) is a separate animated greeting screen — the Dashboard
> (`/dashboard`) is the widget home described here.
